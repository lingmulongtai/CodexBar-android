# CodexBar for Android v0.3.0

This stable release turns CodexBar for Android into a polished, native quota dashboard with safer account linking, customizable widgets, and Android 16 Live Updates.

## Highlights

- Redesigned the app with Material 3 Expressive provider cards, animated quota rings and bars, exact used/remaining values, reset countdowns, and pace forecasts.
- Added adaptive navigation: bottom navigation on phones, a navigation rail on larger displays, and a synchronized two-pane dashboard.
- Added an Android 16 promoted Live Update with quota progress, remaining usage, reset and pace context, plus Refresh and Stop actions. Android 8-15 use a compatible ongoing notification.
- Expanded per-widget controls for provider order, visible quota windows, reset time, freshness, pace, and privacy-aware rendering.
- Added English and Japanese per-app language selection and localized the dashboard, settings, widgets, notifications, Quick Settings tile, and background status messages.

## Safer account connection

- Codex, Gemini, and GitHub Copilot use guided device authorization flows with copyable one-time codes and browser handoff.
- Claude uses the supported `claude setup-token` path with a clearly separated manual fallback.
- Credentials are saved only after validation, encrypted with Android Keystore-backed AES-GCM, excluded from backup and device transfer, and never written to HTTP body logs.
- Google client secrets are rejected because an installed Android app cannot keep them confidential.
- Device authorization opens only approved HTTPS verification origins, bounds polling intervals and provider response sizes, and never follows redirects while exchanging credentials.
- Notification and widget privacy starts redacted until secure settings have loaded; internal widget broadcasts and app navigation are explicitly scoped to CodexBar.

## Reliability and updates

- Registered Hilt's WorkerFactory with WorkManager so widget and monitoring refresh jobs can be constructed after process restarts.
- Manual and widget refreshes now supersede stale or backed-off work, and provider failures render actionable status instead of leaving widgets on `Waiting for data...`.
- Removed the launch-time request for unrestricted battery-optimization exemption; scheduled work now follows Android's normal battery policy.
- Added a trusted release update prompt that accepts only stable releases and the expected `app-release.apk` asset from this repository.
- Pinned release workflow actions to immutable commits and limited signing secrets to the signing step, with automatic keystore cleanup.
- Preserved the application ID, production signing certificate, secure DataStore name, and Keystore alias for in-place upgrades from v0.2.x.
- Updated to Android Gradle Plugin 9.2.1, Gradle 9.4.1, Kotlin 2.3.21, target SDK 36, strict dependency verification, and Node 24-compatible GitHub Actions.

## Install or update

1. Download `app-release.apk` from the assets below.
2. Open it on the Android device and allow installation from the browser or file manager if Android asks.
3. When updating from v0.2.x, install over the existing app. Do not uninstall first if you want to keep linked accounts and settings.

Release assets are signed in the protected production environment and include `SHA256SUMS`, a CycloneDX SBOM, build provenance, and GitHub artifact attestations.

## Verification

- Strict dependency verification, Android lint, unit tests, and debug/release builds are release gates.
- Android 16 / API 36.1 emulator verification covered light and dark dashboards, the large-screen two-pane layout, device-code browser handoff and code copy, promoted ProgressStyle notification actions, widget background refresh construction, and in-place data preservation.
- Screenshots in the README use synthetic quota data and contain no account credentials.

Platform note: Android and device manufacturers decide whether an eligible promoted notification receives the elevated Live Update presentation. When promotion is unavailable, CodexBar keeps the same monitoring session in a standard ongoing notification. Background timing also remains subject to Android battery and scheduling policy; the Refresh action requests an immediate update.
