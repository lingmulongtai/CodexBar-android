# CodexBar for Android v0.4.1

This is a stable patch release, not a beta. It fixes home-screen widgets that could remain on `Loading...` even after the app had already loaded current quota data.

## Widget initialization and launcher compatibility

- Restores standards-compatible widget-provider delivery for launchers, including the lifecycle path used by One UI and other third-party launchers.
- Resolves the Glance widget from the exact App Widget ID allocated by the launcher during configuration.
- Replaces the static `Loading...` provider layout with `Waiting for data...` before Glance composition begins, so a delayed worker cannot leave the launcher on the original XML forever.
- Removes network refresh work from the exported widget receiver; launcher broadcasts can only request local widget rendering.
- Replaces unsupported raw `android.view.View` elements in the widget picker preview with RemoteViews-compatible containers.

## Data synchronization

- Writes the dashboard's freshly fetched presentation snapshot directly to the widget cache and redraws active widgets immediately.
- Avoids waiting for a second WorkManager network request after a successful manual or foreground refresh.
- Renders a useful empty or disconnected state when no linked account remains instead of returning before the widget can leave its loading layout.
- Keeps provider failures isolated from successful dashboard refreshes and preserves the existing cached-data fallback.

## Verification

- Passes dependency verification, Android Lint, unit tests, and debug APK assembly.
- Passes launcher discovery, picker preview, widget allocation, and configuration-activity smoke checks on Android 16 / API 36.
- Verifies the picker no longer reports `Class not allowed to be inflated android.view.View` or displays `Couldn't add widget`.
- Includes regression tests for exact widget-ID resolution, immediate initial rendering, RemoteViews compatibility, receiver network isolation, empty-account rendering, and dashboard-to-widget synchronization.

## Install or update

1. Download `app-release.apk` from the assets below.
2. Optionally compare its SHA-256 digest with `SHA256SUMS`.
3. Install it directly over v0.4.0 to retain linked accounts, widget settings, and history.
4. Remove and add back any widget that was already permanently stuck on the old static loading layout.
5. Open CodexBar and refresh once; the same successful snapshot is now sent to the dashboard, notifications, and widgets.

The APK is signed with the existing production certificate. A signature-conflict warning means the installed app came from a different signer; uninstalling will erase local credentials and settings, so verify the source before doing that.
