# CodexBar for Android v0.3.5

This stable hotfix replaces v0.3.1, which could close immediately during startup. Users of v0.3.1 should update to this release.

## Fixed

- Deferred background refresh scheduling until after Hilt finishes injecting the application-level WorkManager configuration.
- Removed the pre-`Application.onCreate` AndroidX Startup path that could access `HiltWorkerFactory` before initialization.
- Preserved the generated Room database constructor required by WorkManager after R8 minification.
- Removed duplicate refresh scheduling from the main activity while retaining process-start and boot-time refresh restoration.

## Release safety

- The production release workflow installs the signed, minified APK on an Android 16 emulator before publication.
- Publication fails if the app process exits during startup or if Android records a crash for CodexBar.
- The emulator smoke runs in one POSIX shell process with KVM acceleration, using an action pinned to an immutable commit.

## Verification

- Built the minified release APK with strict dependency verification.
- Confirmed the R8 mapping retains `androidx.work.impl.WorkDatabase_Impl` and its public no-argument constructor.
- Completed five consecutive clean installs and cold starts on Android 16 / API 36; every run retained a live app process with an empty crash buffer.
- Passed focused WorkManager, Room keep-rule, release workflow, and workflow action-pinning tests.

## Install or update

1. Download `app-release.apk` from the assets below.
2. Open it on the Android device and allow installation from the browser or file manager if Android asks.
3. Install it over v0.3.1 or an earlier version. Do not uninstall first if you want to keep linked accounts and settings.

Release assets are signed with the existing production certificate and include `SHA256SUMS`, a CycloneDX SBOM, build provenance, and GitHub artifact attestations.
