# CodexBar for Android v0.5.0

This is a stable feature release, not a beta. It expands CodexBar to 15 AI providers, introduces a dedicated Connections destination, and hardens widget rendering for the larger provider catalog.

## More AI providers

- Adds Cursor, z.ai, ZenMux, Kimi Code, ElevenLabs, OpenRouter, Synthetic, Chutes, DeepSeek, Venice, and Moonshot API (International), alongside Claude, Codex, Gemini, and GitHub Copilot.
- Shows rolling subscription windows, character or request quotas, API-key budgets, and exact provider balances according to the data each supported service exposes.
- Keeps providers without a bounded percentage useful by showing exact balance or availability states instead of inventing a progress bar.
- Centralizes provider metadata and repository registration so the dashboard, notifications, widgets, credential storage, and background refresh use the same catalog.

## Dedicated Connections experience

- Adds a top-level **Connections** destination beside Usage and Settings on phones and large screens.
- Moves all account linking and credential management out of Settings, leaving Settings focused on monitoring, refresh behavior, language, and privacy.
- Adds provider search, connection-state filtering, category filtering, a connected-provider summary, collapsible provider cards, and a single credential-removal area.
- Routes dashboard empty states, provider detail actions, widget setup, and Gemini pairing deep links directly to Connections.
- Shares connection state between Connections and Settings so disconnecting or deleting credentials cannot leave another tab with stale account state.

## Security and widget reliability

- Uses credential-safe HTTP clients with fixed HTTPS provider hosts, disabled HTTP body logging, and redirect rejection for manually supplied secrets.
- Validates credentials before Android Keystore-backed encrypted storage and keeps provider failures isolated during multi-provider refresh.
- Prevents widgets from remaining indefinitely on their static loading layout and renders useful states for empty, disconnected, stale, failed, and percentage-free providers.
- Synchronizes dashboard refresh results with widget cache rendering without waiting for a second network refresh.

## Verification

- Passes 211 unit and source-policy tests with no failures.
- Passes strict Gradle dependency verification, Android Lint with no errors, and debug APK assembly.
- Includes provider-specific parsing, fixed-host networking, credential safety, registry coverage, searchable directory, connection navigation, and widget-render regression tests.
- The release workflow separately builds and verifies the production-signed APK and AAB, tests the Gemini companion, runs an Android 16 emulator startup smoke test, generates SBOMs and checksums, and publishes build provenance attestations.

## Install or update

1. Download `app-release.apk` from the assets below.
2. Optionally compare its SHA-256 digest with `SHA256SUMS`.
3. Install it directly over v0.4.1 to retain linked accounts, widget settings, history, and privacy preferences.
4. Open **Connections** to add or review providers, then refresh the Usage dashboard.
5. Gemini users should also update to `CodexBar-Gemini-Companion-v0.5.0.zip` from this Release.

The APK is signed with the existing production certificate. A signature-conflict warning means the installed app came from a different signer; uninstalling erases local credentials and settings, so verify the APK source before doing that.
