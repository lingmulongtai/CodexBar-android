# CodexBar for Android v0.4.0

This is a stable release, not a beta. It restores Gemini monitoring without exporting Google credentials, repairs notification and widget synchronization, and includes the authentication reliability fixes made after v0.3.7.

## Gemini monitoring without copied tokens

- Replaces the unsupported Android Gemini OAuth/device-code path and all direct `cloudcode-pa` access with a private local companion around the official Gemini CLI.
- Removes Gemini access-token, refresh-token, OAuth client-secret, token-refresh, and internal Google quota-request handling from the Android runtime.
- Ships `CodexBar-Gemini-Companion-v0.4.0.zip` with Windows and macOS/Linux launchers, pinned npm dependencies, tests, and its own CycloneDX SBOM.
- Parses only model label, used percentage, reset duration, tier, and freshness from the official CLI `/stats` screen; raw terminal output is discarded and never sent to Android.
- Restricts pairing to numeric local/private addresses and validates the complete snapshot schema, age, values, reset horizon, and duplicate labels.
- Authenticates requests with HMAC-SHA256, rejects stale or replayed nonces, rate-limits clients, and encrypts responses with AES-256-GCM.
- Opens a scanned pairing value in Settings but requires an explicit **Pair & verify companion** action before any key is saved.
- Preserves a working Gemini pairing if a replacement cannot be validated and encrypts the accepted pairing with Android Keystore-backed storage.

## Persistent and live notifications

- Separates the normal persistent notification switch from the time-bounded Live Update session; either can now be enabled or disabled without changing the other.
- Publishes the exact same quota snapshot to dashboard state, persistent notification, and active Live Update after every dashboard refresh, including manual refreshes and partial provider failures.
- Uses a dedicated Live Update notification channel and no longer lets the persistent notification path overwrite or cancel it.
- Corrects progress semantics so bars represent used quota consistently instead of mixing used and remaining percentages.
- Uses Android 16 `Notification.ProgressStyle`, promoted-ongoing metadata, tracker icon, severity colors, countdown, pace, Refresh, and Stop for eligible system and One UI surfaces.
- Keeps the standards-compatible ongoing notification on older Android releases. Android and the device manufacturer still decide whether a notification is promoted into a lock-screen or Now Bar surface.

## Home-screen widgets

- Schedules an immediate quota refresh when a widget is added instead of leaving a fresh widget at `Loading...` until the next periodic worker.
- Recovers from missing or stale widget cache and redraws all active widgets after manual/background refresh.
- Returns the configured widget ID when setup is canceled so the launcher can remove the abandoned host cleanly.
- Retains provider/window/freshness/pace customization and cached display fallback while the network is unavailable.

## Account connection reliability

- Retries transient GitHub Copilot device-flow DNS failures using the same bounded code-expiry behavior already used for Codex.
- Keeps the one-time code visible and copyable before browser navigation, including a paste-ready value for split code fields.
- Retains the v0.3.7 GitHub JSON response fix and Codex DNS retry behavior.

## Security and release integrity

- Keeps Google authentication inside the official CLI and never imports a Gemini CLI token or client secret into Android.
- Purges unusable legacy Gemini OAuth records during secure-store migration.
- Uses logging-disabled token clients, redacted metadata-only API logging, Keystore-backed AES-GCM credential values, and backup/data-transfer exclusions.
- Verifies the desktop companion with syntax checks, socket/protocol tests, replay/tamper tests, and `npm audit` in CI and again in the release workflow.
- Publishes the signed APK/AAB, Android and companion SBOMs, build provenance, `SHA256SUMS`, and GitHub artifact attestations only after signing-certificate verification and an API 36 cold-start smoke test.

## Install or update

1. Download `app-release.apk` from the assets below.
2. Optionally compare its SHA-256 digest with `SHA256SUMS`.
3. Install it over v0.3.7; do not uninstall first if you want to retain linked accounts and settings.
4. Turn off any temporary **Install unknown apps** permission after installation.
5. For Gemini, also download and extract `CodexBar-Gemini-Companion-v0.4.0.zip`, follow its `README.md`, and pair it from the Gemini card in Settings.

The APK is signed with the existing production certificate. A signature-conflict warning means the installed app came from a different signer; uninstalling will erase local credentials and settings, so verify the source before doing that.
